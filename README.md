# home-i-archive

一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定网站或域名，仅作为静态 HTML 文件的存储与展示空间。

## 项目简介

`home-i-archive` 旨在提供一个简洁、通用的 HTML 页面归档方案。你可以将各类独立的网页内容（如个人笔记、工具页面、信息展示页等）以 HTML 文件形式存放于此，并通过 GitHub Pages 或其他静态托管服务直接访问。

## 目录说明

```
.
├── index.html          # 仓库主页，可自定义为导航或说明页
├── pages/              # 存放归档 HTML 页面的目录
│   ├── example-1.html  # 示例归档页面
│   └── example-2.html  # 示例归档页面
├── assets/             # 静态资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：所有独立归档的 HTML 文件放置于此。每个文件应自包含，避免依赖外部资源（除非使用 CDN 或已托管资源）。
- `assets/`：存放页面共用的样式、脚本或图片资源，便于统一管理。
- `index.html`：可作为仓库的入口页面，引导访问者浏览归档列表。

## 页面归档说明

- 每个归档页面应为一个完整的、自包含的 HTML 文档。
- 建议在页面内部添加必要的元信息（如标题、创建日期、简要描述），以便维护和浏览。
- 页面之间应保持独立，避免相互引用。
- 归档内容不涉及具体域名、商业推广或诱导点击行为，仅作为资料展示。

## 使用方式

1. 克隆本仓库到本地。
2. 在 `pages/` 目录下添加你的 HTML 文件。
3. 可选：修改 `index.html` 以添加导航或说明。
4. 推送到 GitHub 并启用 GitHub Pages（设置中选 `main` 分支或 `docs` 文件夹）。
5. 访问 `https://你的用户名.github.io/home-i-archive/` 即可查看归档内容。
